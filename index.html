Naming conventions

General

Use sentence case, letters, numbers, and spaces as default; avoid special characters / \ : * ? " <> |.
Use one separator style when needed (choose and apply consistently);preferred: space-hyphen-space (-)
Use a leading zero by default for numbered items so they sort correctly ( 01–99).

Naming course folders

Course (top-level) folder

Name the parent folder using: [Course abbrev] [Course version]

Level 1 (required) folders

Each course parent folder should contain the following three folders:

00 Course information

01 Facilitation resources

02 Learner resources

Level 2 (module) folders

Use a two-digit sequence number plus module name: [##] [Module name]

Naming files

Use a four-digit number followed by a space and a descriptive title: [####] [Topic title]

Visual design

Whitespace

Whitespace is the empty space between elements.

Alignment

Alignment is arranging elements along a common edge or center to create visual order and enhance readability.
cancel

Misaligned elements result in a disorganized, hard-to-follow design.

check_circle

Consistent alignment of text and images creates an organized, easy-to-read layout.

Visual hierarchy

Visual hierarchy organizes design elements to guide viewer attention in a clear, prioritized order.
cancel

Disrupted element order breaks the natural reading flow, causing confusion.

check_circle

Elements follow a logical flow, directing focus naturally from text to image.

Image quality

Images should be sharp and clear to ensure all elements are easily distinguishable.
cancel

Blurry visuals reduce clarity, making details hard to read and understand.

check_circle

Clear visuals enhance user understanding by making details easy to read.

Image scale

Images should fit the design without losing clarity or proportion.
cancel

Stretched or compressed images cause distortion, reducing professionalism.

check_circle

Properly scaled images maintain clarity and proportion, avoiding distortion.

Presentations

Template and theme

Use the most up-to-date template appropriate for the content you're creating.
When importing content from another slide deck or source, use the "Keep text only" option and reapply the template layout to prevent theme contamination.
Use the preset theme included in the template.
If you make any additions or edits to a template, notify the team so they're aware and can make the necessary updates to any current work.

Remember: Slides support learning, not replace teaching.

Slide layouts

Layout name Purpose

Content density

Each slide should communicate one idea; split complex topics across multiple slides.
Favor more slides with concise content over fewer slides packed with information.
Aim for no more than six bullets per slide and six words per bullet.

Remember: Slides are visual aids, not documents.

Speaker notes

Include speaker notes on every instructional slide.
Do not duplicate what is already visible on the slide.
Do not format speaker notes as a "script."
Write speaker notes with both instructors and students in mind.
Write speaker notes in complete sentences using active voice.
Use speaker notes to provide context for why the slide content matters.

Remember: Speaker notes should expand on, contextualize, or explain the slide content; not repeat it word for word.

Procedures and instructions

Writing steps

Capitalize the first word in each step and end the step with the appropriate end punctuation.
When instructing the user to enter text, provide exactly how the text must be entered, including punctuation.
Bold interface elements; avoid using all uppercase unless it's displayed that way in the UI.

Use a monospace font for user-entered text.

Use parallel structure and consistent verb form.
Orient the user first, then state the action to take.

Grammar and usage

Acronyms and abbreviations

Spell out acronyms on the first reference, followed by the shortened version in parenthesis.
When expressing plural acronyms and abbreviations, use a lowercase "s" without an apostrophe.
Use your best judgment to determine if the acronym or abbreviation is common enough to warrant not defining it.

Capitalization

In general use sentence case; except for proper nouns, Pega-specific terms, and mission titles.
Do not use camel case for UI items; Rule names no longer use camel case.
Use sentence case for placeholder text and field labels in the UI.

Date and time

Use the 12-hour format, followed by a capitalized AM or PM with a space between.
Write full month names and use four-digit years to avoid date confusion. If a date must be expressed numerically, use the format YYYY-MM-DD.

Numbers

Spell out numbers from zero through nine; use numerals for numbers 10 and greater.
Spell out ordinal numbers in text.

Punctuation

Ampersands

Avoid using ampersands (&), except with proper nouns, Pega-specific terms, or UI text.

Commas

Use an Oxford comma for a series of three or more items.

Dashes and hyphens

Use an en dash (–) without spaces to indicate a range of values; think of it like the word through.
Use an em dash (—) to create an interruption in a sentence like a semicolon.
Use a hyphen (-) to represent negative numbers, or to create a compound word; also, use to separate phone, serial, and social security numbers.

Slashes

Avoid "either/or" and "and/or;" use full phrases instead.

Style and voice

Voice and tone

Use a natural, casual voice; aim for a conversational tone that is not overly informal.
Choose active verbs over passive nouns to encourage user action.
Write in the present tense to make the content more understandable.
Avoid passive voice. Meaning the action is performed on the sentence's subject.
Use active voice. Meaning the subject of the sentence performs the action.

Person and perspective

Use second person ("you") when addressing the learner.
Use imperative mood to express direct commands or instructions when writing procedures.
Refer to the product in third person.

Plain language

Use familiar words and short, direct sentences so readers understand it on the first read.

Inclusive language

Use gender-neutral language (they/their as a singular pronoun).
Avoid terms that assume a specific cultural background, physical ability, or level of English fluency.

Generative AI

Using AI for written content

Treat AI as a collaborator, not the author. Use AI for drafts, options, and structure; not final copy.
We are the author of record. We own accuracy, instructional quality, and style compliance for every deliverable.

Responsible use

Do not share sensitive or confidential information about Pega with a Gen AI technology unless you’re using a licensed, Pega-approved Gen AI solution.
Facts and statistics should not come from generated resources due to the risk of hallucinations.
Any facts derived from Gen AI will need to be rigorously researched and reviewed for accuracy.

Remember: AI output is a starting point, always review before including AI-generated content in any deliverable.
