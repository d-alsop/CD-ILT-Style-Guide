Whether you’re developing new, editing existing, or collaborating oninstructor-led content, this guide is your central resource for ensuring clarity, consistency, and quality across all our learning materials.

Naming conventions

General

Use sentence case.

Use letters, numbers, and spaces as default; avoid special characters / \ : * ? " <> |

Use one separator style when needed (choose and apply consistently); preferred: space-hyphen-space (-)

Use a leading zero by default for numbered items so they sort correctly (01–99).

Example: 01 SSA 25 Getting started

Naming course folders

Course (top-level) folder

Name the parent folder using: [Course abbrev] [Course version]

Example: SSA 25

Level 1 (required) folders

Each course parent folder should contain the following three folders:

00 Course Information

01 Facilitation Resources

02 Learner Resources

Level 2 (module) folders

Use a two-digit sequence number plus module name: [##] [Module name]

Example: 01 Foundations of Pega Platform

Naming files

Standard file pattern for instructional assets

Use a four-digit number followed by a space and a descriptive title: [####] [Topic title]

Example: 0101 Introduction to Center-out architecture

Presentations

Template and theme

Use the most up-to-date template appropriate for the content you're creating.

When importing content from another slide deck or source, use the Keep text only option and reapply the template layout to prevent theme contamination.

Use the preset theme incuded in the template.

If you make any additions or edits to a template, notify the team so they're aware and can make any necassary updates.

Remember: Slides support learning, not replace teaching.

Slide layouts

Layout name Purpose

Content density

Each slide should communicate one idea; split complex topics across multiple slides.

Favor more slides with concise content over fewer slides packed with information.

Aim for no more than six bullets per slide and six words per bullet.

Remember: Slides are visual aids, not documents.

Speaker notes

Include speaker notes on every instructional slide.

Do not duplicate what is already visible on the slide.

Do not format speaker notes as a "script."

Write speaker notes with both instructors and students in mind.

Write speaker notes in complete sentences using active voice.

Use speaker notes to provide context for why the slide content matters.

Remember: Speaker notes should expand on, contextualize, or explain the slide content; not repeat it word for word.

Procedures and instructions

Writing steps

Capitalize the first word in each step and end the step with the appropriate end punctuation.

When instructing the user to enter text, provide exactly how the text must be entered, including punctuation.

Bold interface elements; avoid using all uppercase unless it's displayed that way in the UI.

cancel
Click SAVE.
check_circle
Click Save.

Use a monospace font for user-entered text.

cancel
In the Name field, enter Office location.
check_circle
In the Name field, enter Office location.

Use parallel structure and consistent verb form.

cancel
Save the file to your local machine by clicking More and then selecting Download.
check_circle
Click More, and then select Download to save the file to your local machine.

Orient the user first, then state the action to take.

cancel

check_circle
In the navigation pane, click Data.

Style and voice

Voice and tone

Use a natural, casual voice; aim for a conversational tone that is not overly informal.

Choose active verbs over passive nouns to encourage user action.

Write in the present tense to make the content more understandable.

Avoid passive voice. Meaning the action is performed on the sentence's subject.

Use active voice. Meaning the subject of the sentence performs the action.

cancel
The updated materials are published by the curriculum development team.
cancel
Customer service is improved by doing this.
check_circle
The curriculum development team publishes the updated materials.
check_circle
This improves customer service.

Person and perspective

Use second person ("you") when addressing the learner.

cancel
The user can configure the SLA to set response deadlines.
check_circle
You can configure the SLA to set response deadlines.

Use imperative mood for procedures.

cancel
You should click Save to store your changes.
check_circle
Click Save to store your changes.

Refer to the product in third person.

cancel
We create a new Case when you submit the form.
check_circle
Pega Platform creates a new Case when you submit the form.

Plain language

Use familiar words and short, direct sentences so readers understand it on the first read.

cancel
Utilize the application to initiate the process.
check_circle
Use the application to start the process.

Inclusive language

Use gender-neutral language (they/their as a singular pronoun).

Avoid terms that assume a specific cultural background, physical ability, or level of English fluency.

Generative AI

Using AI for written content

Use these guidelines whenever you create, edit, or publish course curriculum, or any other user-facing material with help from generative AI tools.

Core principle: AI output is a starting point, not a finished product. Always review before including AI-generated content in any deliverable.

How to use AI

Treat AI as a collaborator, not the author. Use AI for drafts, options, and structure; not final copy.

You are the author of record. You own accuracy, instructional quality, and style compliance for every deliverable.

What to review in AI output:

  • Check spelling and capitalization of Pega-specific terminology.
  • Confirm technical accuracy.
  • Check for headline case, passive voice, filler phrases, and similar issues.
  • Remove hedging language and corporate buzzwords.
  • Confirm numbered steps orient the user first, then action.

Responsible use

Do not share sensitive or confidential information about Pega with a Gen AI technology unless you’re using a licensed, Pega-approved Gen AI solution.

Facts and statistics should not come from generated resources due to the risk of hallucinations.

Any facts derived from Gen AI will need to be rigorously researched and reviewed for accuracy
